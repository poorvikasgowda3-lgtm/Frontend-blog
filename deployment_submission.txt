Checklist & Quick Steps

1) Supabase (Postgres)
- Create a new project on https://app.supabase.com
- Open SQL Editor and run `database_dump.sql` (file: database_dump.sql)
- In Settings > Database > Connection string, copy the FULL DATABASE_URL

2) Backend (Render or Railway)
- Connect your GitHub repo for `backend/`
- In service Settings > Environment, add `DATABASE_URL` = <Supabase connection string>
- Deploy the service and verify `https://<service>.onrender.com/docs` loads the Swagger UI

3) Frontend (Vercel)
- Connect your GitHub repo for the Next.js frontend
- In Vercel Dashboard > Environment Variables, add `NEXT_PUBLIC_API_URL` = https://<your-backend>.onrender.com
- Deploy the site and open the Vercel URL on a phone to verify data loads
